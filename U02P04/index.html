Punto 1: DOCTYPE

El DOCTYPE indica al navegador qué estándar HTML hemos utilizado. Siempre debemos usar el último estándar, HTML5.

Punto 2: Cierra las etiquetas

Es crucial cerrar todas las etiquetas HTML en el mismo orden en que se abren.

Punto 3: Escribe las etiquetas en minúsculas

Para mejorar la legibilidad del código, es recomendable escribir las etiquetas, atributos y valores en minúsculas.

Punto 4: Atributo ALT en las imágenes

Incluir el atributo alt en las imágenes mejora la accesibilidad de la web.

Punto 5: Usa una indentación coherente

La indentación consistente facilita la lectura del código HTML.

Punto 6: No pongas los estilos en el documento de HTML

Separa el contenido (HTML) del diseño visual (CSS) utilizando archivos CSS externos.

Punto 7: No uses divs para todo

Utiliza las etiquetas estructurales de HTML5 para organizar el contenido en lugar de divs.

Punto 8: No pongas el valor de los atributos booleanos

Para atributos booleanos como disabled, no es necesario especificar un valor.

Punto 9: Indica la codificación de caracteres

Incluye la metaetiqueta que indica la codificación de caracteres en el header de tu página.

Punto 10: Revisa qué etiquetas existen

Conoce la lista de etiquetas HTML para utilizar la etiqueta adecuada para cada elemento.
